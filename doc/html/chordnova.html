ChordNova - Beyond boundaries!

ChordNova is a powerful open-source chord progression analysis plus generation software for multiple operating systems (currently Windows and Linux).

  • Featuring unprecedentedly detailed control over trait parameters of musical chords and progressions, that is way above mainstream softwares, which are only based on triadic chord theories. ChordNova is based on the theory of Parameteric Harmony, supporting 15+ indicators and 40+ detailed parameters through the analysis and generation process.
  • The powerful built-in Chord Analyser/subsititor covers all possible pitch combinations of the entire 12-TET, which brings a leap forward in pop/jazz chord substitution techniques.
  • Provides a range of presets and music examples; supports MIDI file output.
  • Runs on multiple OS (currently available in Windows and Linux) . Now seeking cross-platform developers for iOS/Android/Web/Mac OS.

Overview

The generation process consists of three steps.

The rationale behind is explained in https://www.bilibili.com/video/BV1PL4y1q7vj . Essentially, Chen reckons that by searching a chord that satisfies certain criteria (most importantly, tension) within all chords whose notes are within 4 semintones from its corresponding note in the original chord, we have a pretty good candidate for chord progression.

Homogenization

The initial chord is either generated automatically, or it is directly from GUI input by user. It is then inflated to homogenous size, i.e., Chord::m_max pitches. It is easy to spot that there are \(m\_max - 1 \choose m\_notes\_size\) ways. For example, to inflate a 3-note-chord (C4, E4, G4) to 5=note-chord, there would be 4C2 = 6 ways. This is done by Chord::expand.

Mutation

The result of Chord::expand would then be fed into Chord::set_new_chords, where the homogenous chords would traverse all the mutation possibilities. Currently it would attempt to modify each note up to plus or minus Chord::vl_max semitones.

Filtering

Most of the mutations obtained in step 2 are invalid. For example, the notes might not be monotonically increasing, and the notes might not suffice the requirements of the chord properties, such as chroma, tension, range, etc.. In this step, all invalid possibilities are filtered out.
